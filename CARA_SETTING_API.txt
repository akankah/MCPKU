## CARA MENGATUR API KEY & ENVIRONMENT VARIABLE (WINDOWS)

1. Cara Cek/Edit Environment Variables:
   - Tekan tombol Windows, ketik "env", pilih "Edit the system environment variables".
   - Klik tombol "Environment Variables..." di kanan bawah.
   - Di bagian "User variables for [User]", klik "New...".
   - Masukkan Variable name (contoh: GROQ_API_KEY) dan Variable value (API key Anda).
   - Klik OK di semua jendela.
   - PENTING: Anda harus restart terminal/aplikasi agar perubahan terbaca.

2. Mengatur API Key di Opencode:
   - Opencode menggunakan syntax {env:VAR_NAME}.
   - Jangan masukkan API Key langsung di `opencode.jsonc`.
   - Gunakan format: `"apiKey": "{env:GROQ_API_KEY}"`.

3. Menambahkan Free Model Baru (Contoh di opencode.jsonc):
   - Buka `E:\MCPKU\opencode.jsonc`.
   - Cari blok `"provider"`.
   - Tambahkan model baru di bawah daftar `"models"` provider terkait:
     "nama-model-baru": {}
   - Contoh:
     "models": {
       "llama-3.1-8b-instant": {},
       "model-baru-anda": {}
     }

4. Mengaktifkan/Mengetes Model:
   - Setelah edit, simpan file.
   - Restart opencode (exit session, buka terminal baru).
   - Gunakan perintah untuk mengetes API secara manual jika ragu (bisa pakai PowerShell Invoke-RestMethod).

5. Cara Mengatasi Jika Model Error:
   - Jika muncul error saat memakai model (misal: "Request too large" atau "Unauthorized"):
     a. Cek apakah API Key sudah valid di environment variables.
     b. Cek apakah model masih tersedia di provider tersebut (provider sering mengubah daftar model).
     c. Periksa log error: Jika error "Unauthorized" (401), artinya key salah atau belum set di User Env.
     d. Periksa limitasi: Jika error "Request too large", artinya model tersebut punya batasan context (TPM/RPM). Coba ganti model ke versi yang lebih ringan (misalnya dari -70b ke -8b).
     e. Tes API secara manual: Gunakan perintah PowerShell (Invoke-RestMethod) untuk memastikan apakah server provider memang sedang down atau memang API key-nya yang tidak diterima.
     f. Update config: Jika model sudah tidak didukung provider, hapus baris model tersebut dari file `opencode.jsonc`.

6. Tips Tambahan: Mengetes Koneksi API secara Manual (Jika UI tidak responsif):
   Jika Anda mengalami masalah 
o response di antarmuka opencode, Anda bisa mengetes apakah koneksi ke provider API Anda masih berfungsi melalui terminal PowerShell:

   $headers = @{'Authorization' = 'Bearer YOUR_API_KEY_HERE'} 
   Invoke-RestMethod -Uri 'https://YOUR_API_ENDPOINT_URL/models' -Method Get -Headers $headers

   *Ganti YOUR_API_KEY_HERE dengan API key asli Anda dan YOUR_API_ENDPOINT_URL dengan endpoint provider (misal: https://api.b.ai/v1).*

    Jika perintah ini mengembalikan daftar model, berarti koneksi API di sisi server/jaringan normal, dan masalah kemungkinan besar ada pada cache UI atau konfigurasi file .jsonc Anda.

## FIX MODEL ALIBABA ERROR — TROUBLESHOOTING LENGKAP

### Masalah Umum Alibaba Free Provider

| Error | Penyebab | Solusi |
|---|---|---|
| `401 Unauthorized` | API Key salah / tidak di-set di env | Cek `ALIBABA_API_KEY` di User Environment Variables |
| `403 Forbidden` | Key valid tapi akses ditolak / model tidak diizinkan | Cek dashboard Alibaba Cloud / DashScope console |
| `429 Too Many Requests` | Rate limit TPM/RPM tercapai | Ganti ke model lebih kecil (qwen-turbo → qwen-plus) atau wait |
| `500 Internal Server Error` | Server provider down | Cek status.dashscope.aliyuncs.com, retry nanti |
| `Request too large` | Context window terlalu kecil untuk input | Kurangi input / gunakan model context lebih besar |
| `model_not_found` | Model tidak tersedia di provider | Update `opencode.jsonc` — hapus model deprecated |

### Langkah Debug Otomatis (WAJIB dijalankan sebelum tanya)

```bash
# 1. Test API manual via PowerShell
$headers = @{'Authorization' = "Bearer $env:ALIBABA_API_KEY"}
Invoke-RestMethod -Uri 'https://dashscope.aliyuncs.com/compatible-mode/v1/models' -Method Get -Headers $headers

# 2. Jika gagal → cek env var
echo $env:ALIBABA_API_KEY

# 3. Jika kosong → set di User Env (bukan System)
# Windows: Win+R → sysdm.cpl → Advanced → Environment Variables → User variables → New
# Name: ALIBABA_API_KEY | Value: sk-xxxxxxxxxxxxx

# 4. Restart terminal → test lagi
```

### Auto-Fix Flow (pakai mcp_autofix)

```json
{
  "command": "opencode --model alibaba-free/qwen-turbo \"test\"",
  "workdir": "E:/MCPKU",
  "max_retries": 3,
  "auto_commit": false
}
```
→ `mcp_autofix` akan:
1. Detect error type (401/403/429/500)
2. Search internet untuk fix terbaru
3. Suggest: update key, ganti model, wait rate limit
4. Retry dengan fix

## CARA MENAMBAHKAN SERVER KE TOOL_MANIFEST (ENABLE AUTO PARALLEL EXECUTION)

### 1. Buka file manifest
```bash
E:\MCPKU\mcp_manifest.py
```

### 2. Tambahkan ToolEntry untuk setiap fungsi server
Gunakan helper `_entry(name, module, func_name, **kwargs)` di blok `TOOL_MANIFEST`:

```python
_entry("tool_name", "mcp_module", "function_name",
       category="kategori",          # research, debug, files, git, github, perf, vector, time, plan, think, bash, refactor
       description="Deskripsi singkat",
       requires_network=True/False,  # True kalau butuh internet
       dangerous=True/False,         # True kalau destructive (write/delete)
       timeout_suggestion=30,        # Detik, hint untuk model
       parallel_ok=True/False)       # True = bisa jalan parallel dengan tool lain
```

### 3. Contoh lengkap (mcp_github):
```python
# Di section GITHUB
_entry("github_list_issues", "mcp_github", "list_issues",
       category="github", description="List issues", requires_network=True, timeout_suggestion=15),
_entry("github_create_issue", "mcp_github", "create_issue",
       category="github", description="Create issue", requires_network=True, dangerous=True, timeout_suggestion=15),
_entry("github_search_code", "mcp_github", "search_code",
       category="github", description="Search code in repo", requires_network=True, timeout_suggestion=15, parallel_ok=True),
```

### 4. Auto Parallel Execution — Cara Kerja
Setelah di manifest dengan `parallel_ok=True`, model bisa memanggil **multiple tools sekaligus** dalam satu batch:

```json
// Satu round-trip parallel (cepat)
parallel([
  github_search_code(query="error handler"),
  web_search_stackoverflow("python ModuleNotFoundError"),
  diagnostics_classify_error("ModuleNotFoundError: No module named 'requests'"),
  research_query("how to fix missing imports in Python")
])
```

**Syarat parallel:**
- Tool harus di `TOOL_MANIFEST` dengan `parallel_ok=True`
- Tool tidak bergantung output tool lain (independen)
- Kategori berbeda/sama tapi fungsi independen

### 5. Category yang support parallel (recommended):
| Category | Tools yang sebaiknya parallel_ok=True |
|---|---|
| research | web_search_*, search_*, research_*, browser_fetch |
| debug | diagnostics_*, autofix_search_kb, autofix_kb_stats |
| github | github_search_code, github_search_issues |

### 7. Tips Best Practice
- **Selalu set `timeout_suggestion`** untuk tool network (10-30s)
- **`dangerous=True`** untuk write/delete/mutate operations
- **`requires_network=True`** untuk tool butuh internet
- **`parallel_ok=True`** hanya untuk read-only / search / independent tools
- **Category konsisten** agar mudah discovery: research, debug, files, git, github, perf, vector, time, plan, think, bash, refactor

```bash
# Test read 10 file parallel vs sequential

import asyncio, time
from mcp_filesystem import read_file

async def test():
    files = [f'E:/MCPKU/test_{i}.txt' for i in range(10)]
    # Sequential
    start = time.time()
    for f in files:
        try: await read_file(f)
        except: pass
    print(f'Sequential: {time.time()-start:.2f}s')
    
    # Parallel
    start = time.time()
    await asyncio.gather(*[read_file(f) for f in files], return_exceptions=True)
    print(f'Parallel: {time.time()-start:.2f}s')

asyncio.run(test())
"
```
